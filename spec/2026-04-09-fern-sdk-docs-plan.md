# Fern-driven SDKs and Docs — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the handwritten TypeScript and Python SDKs with Fern-generated clients driven by the remote OpenAPI spec in `github.com/superserve-ai/sandbox`, and replace Mintlify docs with Fern Docs.

**Architecture:** A single `fern/` directory at the repo root holds Fern config that references the remote spec via HTTP URL. Two generators write into the existing `packages/sdk/` and `packages/python-sdk/` workspace packages. `.fernignore` files preserve hand-maintained manifests. CI is manual-only via `workflow_dispatch`; all operations are documented in the root README.

**Tech Stack:** Fern CLI (OSS), Bun workspaces, Turborepo, tsup (TS build), hatchling + uv (Python build), GitHub Actions.

**Spec:** `spec/2026-04-09-fern-sdk-docs-design.md`.

**Context for the implementer:**
- The Superserve monorepo uses Bun workspaces (`bun install` at root). Never `cd` into a package to add dependencies — always use `bun add <dep> --filter <pkg>` from the root.
- The current handwritten TS SDK has two entry points: `.` and `./react`. The `./react` entry is being **dropped entirely**. No React hooks in the new SDK.
- The existing `packages/sdk/src/` contains: `client.ts`, `session.ts`, `stream.ts`, `sse.ts`, `types.ts`, `errors.ts`, `index.ts`, and `react/`. All of it is going away.
- The existing `packages/python-sdk/src/superserve/` contains handwritten `App`, `Session`, `Stream` classes. All of it is going away.
- Only one file in the monorepo references `@superserve/sdk` in actual code paths: `apps/console/src/app/(dashboard)/get-started/page.tsx`. And even there, the reference is inside a **string literal** for display on the onboarding page — not a real import. Other matches (README, CLAUDE.md, bun.lock, package.json itself) are metadata/docs, not code.
- The remote OpenAPI spec URL is `https://raw.githubusercontent.com/superserve-ai/sandbox/refs/heads/main/api/openapi.yaml`. Fern fetches it on each `fern generate` run.
- All work happens on the existing `jeet/fern-sdk-docs` branch.
- Commit frequently — one commit per task minimum. Use single-line commit messages (per project CLAUDE.md), no Co-Authored-By trailers.

```yaml
name: Release SDKs

on:
  workflow_dispatch:
    inputs:
      package:
        description: "Which package to publish"
        required: true
        type: choice
        options:
          - ts
          - python
          - both
      version:
        description: "New version (semver, e.g. 0.2.1)"
        required: true
        type: string

jobs:
  release-ts:
    if: ${{ inputs.package == 'ts' || inputs.package == 'both' }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: 1.3.5

      - name: Install
        run: bun install --frozen-lockfile

      - name: Bump version
        run: |
          cd packages/sdk
          node -e "const fs=require('fs'); const p=require('./package.json'); p.version='${{ inputs.version }}'; fs.writeFileSync('./package.json', JSON.stringify(p, null, 2) + '\n');"

      - name: Build
        run: bunx turbo run build --filter=@superserve/sdk

      - name: Publish to npm
        run: cd packages/sdk && bun publish --access public
        env:
          NPM_CONFIG_TOKEN: ${{ secrets.NPM_TOKEN }}

      - name: Commit version bump
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
          git add packages/sdk/package.json
          git commit -m "release: @superserve/sdk@${{ inputs.version }}"
          git tag "@superserve/sdk@${{ inputs.version }}"
          git push
          git push --tags

  release-python:
    if: ${{ inputs.package == 'python' || inputs.package == 'both' }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup uv
        uses: astral-sh/setup-uv@v4

      - name: Bump version
        run: |
          cd packages/python-sdk
          python3 -c "import re; p=open('pyproject.toml').read(); p=re.sub(r'version = \"[^\"]+\"', 'version = \"${{ inputs.version }}\"', p, count=1); open('pyproject.toml','w').write(p)"

      - name: Build
        run: cd packages/python-sdk && uv build

      - name: Publish to PyPI
        run: cd packages/python-sdk && uv publish
        env:
          UV_PUBLISH_TOKEN: ${{ secrets.PYPI_TOKEN }}

      - name: Commit version bump
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
          git add packages/python-sdk/pyproject.toml
          git commit -m "release: superserve-py@${{ inputs.version }}"
          git tag "superserve-py@${{ inputs.version }}"
          git push
          git push --tags
```
