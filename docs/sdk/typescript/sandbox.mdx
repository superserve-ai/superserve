## Import

```typescript
import { Sandbox } from "@superserve/sdk"
```

## Authentication

All factory methods accept `apiKey` and `baseUrl`, or read from environment variables:

```bash
export SUPERSERVE_API_KEY=ss_live_...
export SUPERSERVE_BASE_URL=https://api.superserve.ai  # optional
```

## Factory methods

### `Sandbox.create(options)`

Create a new sandbox. The call is synchronous ŌĆö when it returns, the sandbox is `active` and ready to run commands and file operations.

```typescript
const sandbox = await Sandbox.create({
  name: "my-sandbox",
  timeoutSeconds: 3600,
  metadata: { env: "prod", owner: "agent-7" },
  envVars: { OPENAI_API_KEY: "sk-..." },
  network: {
    allowOut: ["api.openai.com", "*.github.com"],
    denyOut: ["0.0.0.0/0"],
  },
})

const result = await sandbox.commands.run("echo hello")
```

**Options:**

| Field | Type | Description |
|---|---|---|
| `name` | `string` | **Required.** Human-readable sandbox name. |
| `timeoutSeconds` | `number` | Cap on how long the sandbox can stay `active` before it's automatically paused (max 604800). |
| `metadata` | `Record<string, string>` | String tags attached at creation. |
| `envVars` | `Record<string, string>` | Environment variables injected into every process. |
| `network` | `NetworkConfig` | Egress allow/deny rules. |
| `apiKey` | `string` | Overrides `SUPERSERVE_API_KEY` env var. |
| `baseUrl` | `string` | Overrides `SUPERSERVE_BASE_URL` env var. |
| `signal` | `AbortSignal` | Abort the creation request. |

### `Sandbox.connect(sandboxId, options?)`

Connect to an existing sandbox by ID. Fetches info and access token.

```typescript
const sandbox = await Sandbox.connect("sandbox-uuid")
```

### `Sandbox.list(options?)`

List all sandboxes for the authenticated team.

```typescript
const all = await Sandbox.list()
const prod = await Sandbox.list({ metadata: { env: "prod" } })
```

Returns `SandboxInfo[]`.

### `Sandbox.get(sandboxId, options?)`

Get sandbox info by ID without creating an instance.

```typescript
const info = await Sandbox.get("sandbox-uuid")
```

Returns `SandboxInfo`.

### `Sandbox.killById(sandboxId, options?)`

Delete a sandbox by ID. Idempotent ŌĆö deleting an already-deleted sandbox is a no-op.

```typescript
await Sandbox.killById("sandbox-uuid")
```

## Instance properties

| Property | Type | Description |
|---|---|---|
| `id` | `string` | Unique sandbox ID (UUID). |
| `name` | `string` | Human-readable sandbox name. |
| `status` | `SandboxStatus` | Status at construction time. Call `getInfo()` to refresh. |
| `metadata` | `Record<string, string>` | Tags at construction time. |
| `commands` | `Commands` | [Commands](/sdk/typescript/commands) sub-module. |
| `files` | `Files` | [Files](/sdk/typescript/files) sub-module (rebuilt transparently after `resume()`). |

## Instance methods

### `sandbox.getInfo()`

Fetch the latest info from the API. Returns a fresh `SandboxInfo` ŌĆö the instance's own `status`/`metadata` are snapshots and are not mutated.

```typescript
const info = await sandbox.getInfo()
console.log(info.status)  // "active" or "idle"
```

### `sandbox.pause()`

Pause the sandbox. All state (memory, processes, files) is preserved. Status transitions to `idle`.

```typescript
await sandbox.pause()
```

Returns `Promise<void>`.

### `sandbox.resume()`

Resume a paused sandbox. Status transitions back to `active`. The access token is rotated on the backend; the SDK updates the `files` sub-module transparently.

```typescript
await sandbox.resume()
```

Returns `Promise<void>`.

### `sandbox.kill()`

Delete the sandbox and all its resources. Idempotent.

```typescript
await sandbox.kill()
```

### `sandbox.update(options)`

Partially update the sandbox. Only `metadata` and `network` are patchable.

```typescript
await sandbox.update({
  metadata: { env: "staging" },
  network: { allowOut: ["api.openai.com"] },
})
```

### Auto-cleanup with `await using`

```typescript
{
  await using sandbox = await Sandbox.create({ name: "scoped" })
  await sandbox.commands.run("echo hello")
  // sandbox.kill() runs automatically when scope exits (even on throw)
}
```

## Types

### `SandboxStatus`

```typescript
type SandboxStatus = "active" | "idle"
```

A sandbox is `active` when running, `idle` after `pause()`. Deletion removes the sandbox ŌĆö the API returns `404` afterward.

### `SandboxInfo`

```typescript
interface SandboxInfo {
  id: string
  name: string
  status: SandboxStatus
  vcpuCount: number
  memoryMib: number
  createdAt: Date
  timeoutSeconds?: number
  network?: NetworkConfig
  metadata: Record<string, string>
}
```

### `NetworkConfig`

```typescript
interface NetworkConfig {
  allowOut?: string[]  // CIDRs or domains to allow
  denyOut?: string[]   // CIDRs to deny (use "0.0.0.0/0" to deny all)
}
```

## Errors

All errors extend `SandboxError`:

```typescript
import {
  SandboxError,
  AuthenticationError,   // 401
  ValidationError,       // 400
  NotFoundError,         // 404
  ConflictError,         // 409 ŌĆö invalid state for operation
  TimeoutError,          // request timed out
  ServerError,           // 500
} from "@superserve/sdk"

try {
  await sandbox.pause()
} catch (err) {
  if (err instanceof ConflictError) {
    // Sandbox is not in a pausable state
  }
}
```
