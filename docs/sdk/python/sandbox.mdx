## Import

```python
from superserve import Sandbox

# Async variant
from superserve import AsyncSandbox
```

Both `Sandbox` and `AsyncSandbox` share the same method names and signatures. Async methods are awaitable; everything else is identical.

## Authentication

All factory methods accept `api_key` and `base_url`, or read from environment variables:

```bash
export SUPERSERVE_API_KEY=ss_live_...
export SUPERSERVE_BASE_URL=https://api.superserve.ai  # optional
```

## Factory methods

### `Sandbox.create(...)`

Create a new sandbox. The call is synchronous â€” when it returns, the sandbox is `active` and ready to run commands and file operations.

```python
from superserve import Sandbox, NetworkConfig

sandbox = Sandbox.create(
    name="my-sandbox",
    timeout_seconds=3600,
    metadata={"env": "prod", "owner": "agent-7"},
    env_vars={"OPENAI_API_KEY": "sk-..."},
    network=NetworkConfig(
        allow_out=["api.openai.com", "*.github.com"],
        deny_out=["0.0.0.0/0"],
    ),
)

result = sandbox.commands.run("echo hello")
```

**Parameters:**

| Field | Type | Description |
|---|---|---|
| `name` | `str` | **Required.** Human-readable sandbox name. |
| `timeout_seconds` | `int` | Cap on how long the sandbox can stay `active` before it's automatically paused (max 604800). |
| `metadata` | `dict[str, str]` | String tags attached at creation. |
| `env_vars` | `dict[str, str]` | Environment variables injected into every process. |
| `network` | `NetworkConfig` | Egress allow/deny rules. |
| `api_key` | `str` | Overrides `SUPERSERVE_API_KEY` env var. |
| `base_url` | `str` | Overrides `SUPERSERVE_BASE_URL` env var. |

### `Sandbox.connect(sandbox_id, ...)`

Connect to an existing sandbox by ID. Fetches info and access token.

```python
sandbox = Sandbox.connect("sandbox-uuid")
```

### `Sandbox.list(...)`

List all sandboxes for the authenticated team.

```python
all_boxes = Sandbox.list()
prod_boxes = Sandbox.list(metadata={"env": "prod"})
```

Returns `list[SandboxInfo]`.

### `Sandbox.get(sandbox_id, ...)`

Get sandbox info by ID without creating an instance.

```python
info = Sandbox.get("sandbox-uuid")
```

Returns `SandboxInfo`.

### `Sandbox.kill_by_id(sandbox_id, ...)`

Delete a sandbox by ID. Idempotent â€” deleting an already-deleted sandbox is a no-op.

```python
Sandbox.kill_by_id("sandbox-uuid")
```

## Instance attributes

| Attribute | Type | Description |
|---|---|---|
| `id` | `str` | Unique sandbox ID (UUID). |
| `name` | `str` | Human-readable sandbox name. |
| `status` | `SandboxStatus` | Status at construction time. Call `get_info()` to refresh. |
| `metadata` | `dict[str, str]` | Tags at construction time. |
| `commands` | `Commands` | [Commands](/sdk/python/commands) sub-module. |
| `files` | `Files` | [Files](/sdk/python/files) sub-module (rebuilt transparently after `resume()`). |

## Instance methods

### `sandbox.get_info()`

Fetch the latest info from the API. Returns a fresh `SandboxInfo` â€” instance attributes (`status`, `metadata`) are snapshots and are not mutated.

```python
info = sandbox.get_info()
print(info.status.value)  # "active" or "idle"
```

### `sandbox.pause()`

Pause the sandbox. All state (memory, processes, files) is preserved. Status transitions to `idle`.

```python
sandbox.pause()
```

Returns `None`.

### `sandbox.resume()`

Resume a paused sandbox. Status transitions back to `active`. The access token is rotated on the backend; the SDK updates the `files` sub-module transparently.

```python
sandbox.resume()
```

Returns `None`.

### `sandbox.kill()`

Delete the sandbox and all its resources. Idempotent.

```python
sandbox.kill()
```

### `sandbox.update(...)`

Partially update the sandbox. Only `metadata` and `network` are patchable.

```python
sandbox.update(
    metadata={"env": "staging"},
    network=NetworkConfig(allow_out=["api.openai.com"]),
)
```

### Auto-cleanup with `with`

```python
with Sandbox.create(name="scoped") as sandbox:
    result = sandbox.commands.run("echo hello")
# sandbox.kill() runs automatically (even on exception)
```

## Async usage

`AsyncSandbox` has the same API with awaitable methods and `async with` support:

```python
import asyncio
from superserve import AsyncSandbox

async def main():
    async with await AsyncSandbox.create(name="async-example") as sandbox:
        result = await sandbox.commands.run("echo hello")
        print(result.stdout)

asyncio.run(main())
```

## Types

### `SandboxStatus`

```python
class SandboxStatus(str, Enum):
    ACTIVE = "active"
    IDLE = "idle"
```

A sandbox is `ACTIVE` when running, `IDLE` after `pause()`. Deletion removes the sandbox â€” the API returns `404` afterward.

### `SandboxInfo`

```python
class SandboxInfo(BaseModel):
    id: str
    name: str
    status: SandboxStatus
    vcpu_count: int
    memory_mib: int
    created_at: datetime
    timeout_seconds: int | None = None
    network: NetworkConfig | None = None
    metadata: dict[str, str]
```

### `NetworkConfig`

```python
class NetworkConfig(BaseModel):
    allow_out: list[str] | None = None  # CIDRs or domains to allow
    deny_out: list[str] | None = None   # CIDRs to deny
```

## Errors

All errors extend `SandboxError`:

```python
from superserve import (
    SandboxError,
    AuthenticationError,     # 401
    ValidationError,         # 400
    NotFoundError,           # 404
    ConflictError,           # 409 â€” invalid state for operation
    SandboxTimeoutError,     # request timed out (does not shadow builtin TimeoutError)
    ServerError,             # 500
)

try:
    sandbox.pause()
except ConflictError:
    # Sandbox is not in a pausable state
    pass
```
