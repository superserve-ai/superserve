---
title: "Authentication"
description: "API key authentication for Superserve"
---

All Superserve API requests require an API key passed in the `X-API-Key` header. The base URL is `https://api.agentbox.dev/v1`.

## Get an API key

Generate an API key from the [Superserve Console](https://console.superserve.ai). Keys are prefixed with `ss_`.

## CLI authentication

Log in with the CLI to store your API key locally:

```bash
superserve login
```

Or pass the key directly (useful for CI/CD):

```bash
superserve login --api-key ss_your_api_key
```

Credentials are stored at `~/.superserve/auth.json` with owner-only permissions (`0600`).

## Log out

```bash
superserve logout
```

This removes `~/.superserve/auth.json`.

## Environment variable

Set the `SUPERSERVE_API_KEY` environment variable to authenticate without logging in. The CLI and SDKs both check this variable automatically.

```bash
export SUPERSERVE_API_KEY=ss_your_api_key
```

This is the recommended approach for CI/CD pipelines and server-side applications.

## SDK authentication

Pass the API key when creating a client:

<CodeGroup>

```typescript TypeScript
import { Superserve } from "@superserve/sdk"

const client = new Superserve({ apiKey: "ss_your_api_key" })
```

```python Python
from superserve import Superserve

client = Superserve(api_key="ss_your_api_key")
```

</CodeGroup>

If no `apiKey` is provided, both SDKs fall back to the `SUPERSERVE_API_KEY` environment variable.

## Direct API usage

Include the `X-API-Key` header in requests to the API:

```bash
curl https://api.agentbox.dev/v1/vms \
  -H "X-API-Key: ss_your_api_key"
```

## Key precedence

1. Explicit `apiKey` parameter (SDK) or `--api-key` flag (CLI)
2. `SUPERSERVE_API_KEY` environment variable
3. Stored credentials at `~/.superserve/auth.json`
