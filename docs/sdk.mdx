---
title: "TypeScript SDK"
description: "TypeScript client for managing VMs, executing commands, and working with checkpoints"
---

Manage Superserve VMs programmatically from TypeScript and JavaScript.

```bash
bun add @superserve/sdk
```

```typescript
import { Superserve } from "@superserve/sdk"

const client = new Superserve({ apiKey: "ss_your_api_key" })
```

<Info>
  Get an API key from the [Superserve Console](https://console.superserve.ai) or set the `SUPERSERVE_API_KEY` environment variable.
</Info>

### Platform support

- **Node.js** 18+
- Ships as both **ESM** and **CJS**

---

## VMs

Create, list, and manage VMs.

```typescript
// Create a VM
const vm = await client.vms.create({
  name: "my-sandbox",
  image: "ubuntu-22.04",
  vcpu: 2,
  memory: 512,
})

console.log(vm.id)     // "vm_a1b2c3d4"
console.log(vm.status) // "RUNNING"

// List VMs
const vms = await client.vms.list()
const running = await client.vms.list({ status: "RUNNING" })

// Get a specific VM
const vm = await client.vms.get("vm_a1b2c3d4")

// Stop, start, sleep, wake
await client.vms.stop("vm_a1b2c3d4")
await client.vms.start("vm_a1b2c3d4")
await client.vms.sleep("vm_a1b2c3d4")
await client.vms.wake("vm_a1b2c3d4")

// Delete
await client.vms.delete("vm_a1b2c3d4")
```

**CreateVMOptions:**

| Field | Type | Description |
|-------|------|-------------|
| `name` | `string` | VM name (required) |
| `image` | `string` | Base image (required, e.g. `"ubuntu-22.04"`) |
| `vcpu` | `number?` | Number of vCPUs (default: 2) |
| `memory` | `number?` | Memory in MB (default: 512) |

**VM:**

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | VM ID (e.g. `"vm_a1b2c3d4"`) |
| `name` | `string` | VM name |
| `image` | `string` | Base image |
| `status` | `string` | `"CREATING"`, `"RUNNING"`, `"STOPPED"`, `"SLEEPING"`, or `"DEAD"` |
| `vcpu` | `number` | Number of vCPUs |
| `memory` | `number` | Memory in MB |
| `createdAt` | `string` | ISO 8601 timestamp |

## Exec

Run commands inside a VM.

```typescript
const result = await client.exec("vm_a1b2c3d4", {
  command: "echo hello world",
})

console.log(result.stdout)   // "hello world\n"
console.log(result.exitCode) // 0
```

With timeout and streaming:

```typescript
const result = await client.exec("vm_a1b2c3d4", {
  command: "python3 /app/train.py",
  timeout: 300,
  stream: true,
  onStdout: (chunk) => process.stdout.write(chunk),
  onStderr: (chunk) => process.stderr.write(chunk),
})
```

**ExecOptions:**

| Field | Type | Description |
|-------|------|-------------|
| `command` | `string` | Command to execute (required) |
| `timeout` | `number?` | Timeout in seconds |
| `stream` | `boolean?` | Stream output in real time |
| `onStdout` | `(chunk: string) => void` | Callback for stdout chunks (when streaming) |
| `onStderr` | `(chunk: string) => void` | Callback for stderr chunks (when streaming) |

**ExecResult:**

| Field | Type | Description |
|-------|------|-------------|
| `stdout` | `string` | Standard output |
| `stderr` | `string` | Standard error |
| `exitCode` | `number` | Process exit code |

## Files

Upload files to and download files from a VM.

```typescript
// Upload a file
const data = Buffer.from("print('hello')")
await client.files.upload("vm_a1b2c3d4", "/app/script.py", data)

// Upload from a file path
import { readFileSync } from "fs"
const buffer = readFileSync("./model.bin")
await client.files.upload("vm_a1b2c3d4", "/app/model.bin", buffer)

// Download a file
const content = await client.files.download("vm_a1b2c3d4", "/app/results.json")
```

## Checkpoints

Snapshot VM state for rollback or forking.

```typescript
// Create a checkpoint
const cp = await client.checkpoints.create("vm_a1b2c3d4", {
  name: "baseline",
})

console.log(cp.id) // "cp_x7y8z9"

// List checkpoints
const checkpoints = await client.checkpoints.list("vm_a1b2c3d4")

// Delete a checkpoint
await client.checkpoints.delete("vm_a1b2c3d4", "cp_x7y8z9")
```

**Checkpoint:**

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Checkpoint ID |
| `name` | `string` | Checkpoint name |
| `vmId` | `string` | Parent VM ID |
| `createdAt` | `string` | ISO 8601 timestamp |

## Rollback

Restore a VM to a previous checkpoint.

```typescript
await client.rollback("vm_a1b2c3d4", {
  checkpointId: "cp_x7y8z9",
})
```

**RollbackOptions:**

| Field | Type | Description |
|-------|------|-------------|
| `checkpointId` | `string?` | Checkpoint ID to roll back to |
| `name` | `string?` | Checkpoint name to roll back to |
| `minutesAgo` | `number?` | Roll back to nearest checkpoint from N minutes ago |

Provide exactly one of `checkpointId`, `name`, or `minutesAgo`.

## Fork

Clone a VM into independent copies.

```typescript
const forks = await client.fork("vm_a1b2c3d4", { count: 3 })

console.log(forks) // [{ id: "vm_fork_001", ... }, { id: "vm_fork_002", ... }, { id: "vm_fork_003", ... }]
```

Fork from a specific checkpoint:

```typescript
const forks = await client.fork("vm_a1b2c3d4", {
  count: 2,
  fromCheckpoint: "cp_x7y8z9",
})
```

**ForkOptions:**

| Field | Type | Description |
|-------|------|-------------|
| `count` | `number?` | Number of forks (default: 1) |
| `fromCheckpoint` | `string?` | Fork from a specific checkpoint instead of current state |

## Configuration

```typescript
const client = new Superserve({
  apiKey: "ss_your_api_key",   // or set SUPERSERVE_API_KEY env var
  baseUrl: "https://...",       // default: https://api.agentbox.dev/v1
  timeout: 30000,               // ms, default: 30000
})
```

## Errors

All errors extend `SuperserveError`. API errors throw `APIError` with a status code.

```typescript
import { APIError, SuperserveError } from "@superserve/sdk"

try {
  await client.vms.get("vm_nonexistent")
} catch (e) {
  if (e instanceof APIError) {
    console.error(e.status)   // 404
    console.error(e.message)  // "VM not found"
  }
}
```

| Status | Meaning |
|--------|---------|
| `401` | Invalid or missing API key |
| `404` | VM or checkpoint not found |
| `409` | VM is in an incompatible state for the requested operation |
| `422` | Invalid request parameters |
