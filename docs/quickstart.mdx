<Info>
  **You need:** A [Superserve account](https://console.superserve.ai) and an API key.
</Info>

## Install the CLI

```bash
bun add -g @superserve/cli
```

## Authenticate

```bash
superserve login
```

Or pass an API key directly:

```bash
superserve login --api-key ss_your_api_key
```

## Create a VM

```bash
superserve vm create --name my-sandbox --image ubuntu-22.04
```

```
Created VM vm_a1b2c3d4 (ubuntu-22.04, 2 vCPU, 512 MB)
Status: RUNNING
```

The VM boots in under 200ms and is ready to use immediately.

## Run a command

```bash
superserve exec vm_a1b2c3d4 "echo hello world"
```

```
hello world
```

## Upload a file

```bash
superserve files upload vm_a1b2c3d4 ./script.py /app/script.py
```

Then run it:

```bash
superserve exec vm_a1b2c3d4 "python3 /app/script.py"
```

## Create a checkpoint

Save the current VM state so you can roll back later:

```bash
superserve checkpoint create vm_a1b2c3d4 --name baseline
```

```
Created checkpoint cp_x7y8z9 "baseline"
```

## Fork the VM

Create independent copies of the VM from the current state:

```bash
superserve fork vm_a1b2c3d4 --count 2
```

```
Forked vm_a1b2c3d4 into 2 VMs:
  vm_fork_001
  vm_fork_002
```

Each fork is a fully independent VM with the same filesystem and memory state as the original.

## Roll back

Restore the VM to a previous checkpoint:

```bash
superserve rollback vm_a1b2c3d4 --name baseline
```

The VM is instantly restored to the exact state when the checkpoint was created.

## Clean up

```bash
superserve vm delete vm_a1b2c3d4
```
