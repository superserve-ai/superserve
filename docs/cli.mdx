---

## `superserve login`

Authenticate the CLI. Stores credentials at `~/.superserve/auth.json`.

```bash
superserve login
superserve login --api-key ss_your_api_key
```

| Flag | Description |
|------|-------------|
| `--api-key <key>` | Authenticate with an API key directly |

---

## `superserve logout`

Remove stored credentials.

```bash
superserve logout
```

---

## `superserve vm create`

Create a new VM.

```bash
superserve vm create --name <name> --image <image> [--vcpu <n>] [--memory <mb>]
```

| Flag | Description |
|------|-------------|
| `--name <name>` | VM name (required) |
| `--image <image>` | Base image (required, e.g. `ubuntu-22.04`) |
| `--vcpu <n>` | Number of vCPUs (default: 2) |
| `--memory <mb>` | Memory in MB (default: 512) |

```bash
superserve vm create --name my-sandbox --image ubuntu-22.04
superserve vm create --name heavy-compute --image ubuntu-22.04 --vcpu 4 --memory 2048
```

---

## `superserve vm list`

List all VMs.

```bash
superserve vm list [--status <status>]
```

| Flag | Description |
|------|-------------|
| `--status <status>` | Filter by status (`RUNNING`, `STOPPED`, `SLEEPING`, `CREATING`) |

---

## `superserve vm get`

Get details for a specific VM.

```bash
superserve vm get <vm_id>
```

---

## `superserve vm delete`

Delete a VM permanently.

```bash
superserve vm delete <vm_id> [--force]
```

| Flag | Description |
|------|-------------|
| `--force` | Skip confirmation prompt |

---

## `superserve vm stop`

Stop a running VM.

```bash
superserve vm stop <vm_id>
```

---

## `superserve vm start`

Start a stopped VM.

```bash
superserve vm start <vm_id>
```

---

## `superserve vm sleep`

Suspend a VM to disk. The VM can be woken later with its full state restored.

```bash
superserve vm sleep <vm_id>
```

---

## `superserve vm wake`

Wake a sleeping VM. Restores the full VM state from disk.

```bash
superserve vm wake <vm_id>
```

---

## `superserve exec`

Execute a command inside a VM.

```bash
superserve exec <vm_id> <command> [--timeout <seconds>] [--stream]
```

| Flag | Description |
|------|-------------|
| `--timeout <seconds>` | Command timeout in seconds |
| `--stream` | Stream stdout/stderr in real time |

```bash
superserve exec vm_abc123 "echo hello"
superserve exec vm_abc123 "python3 /app/train.py" --stream --timeout 300
```

---

## `superserve ssh`

Open an interactive SSH session to a VM.

```bash
superserve ssh <vm_id>
```

---

## `superserve files upload`

Upload a local file to a VM.

```bash
superserve files upload <vm_id> <local_path> <remote_path>
```

```bash
superserve files upload vm_abc123 ./script.py /app/script.py
superserve files upload vm_abc123 ./data.csv /tmp/data.csv
```

---

## `superserve files download`

Download a file from a VM.

```bash
superserve files download <vm_id> <remote_path> [local_path]
```

If `local_path` is omitted, the file is written to the current directory with its remote filename.

```bash
superserve files download vm_abc123 /app/results.json
superserve files download vm_abc123 /app/results.json ./output.json
```

## `superserve checkpoint create`

Create a checkpoint of a VM's current state.

```bash
superserve checkpoint create <vm_id> --name <name>
```

| Flag | Description |
|------|-------------|
| `--name <name>` | Checkpoint name (required) |

```bash
superserve checkpoint create vm_abc123 --name baseline
```


| Flag | Description |
|------|-------------|
| `--force` | Skip confirmation prompt |

---

## `superserve rollback`

Restore a VM to a previous checkpoint.

```bash
superserve rollback <vm_id> [--checkpoint <checkpoint_id>] [--name <name>] [--minutes-ago <n>]
```

| Flag | Description |
|------|-------------|
| `--checkpoint <id>` | Roll back to a specific checkpoint ID |
| `--name <name>` | Roll back to a checkpoint by name |
| `--minutes-ago <n>` | Roll back to the nearest checkpoint from N minutes ago |

```bash
superserve rollback vm_abc123 --name baseline
superserve rollback vm_abc123 --checkpoint cp_x7y8z9
superserve rollback vm_abc123 --minutes-ago 30
```

---

## `superserve fork`

Fork a VM into one or more independent copies.

```bash
superserve fork <vm_id> [--count <n>] [--from-checkpoint <checkpoint_id>]
```

| Flag | Description |
|------|-------------|
| `--count <n>` | Number of forks to create (default: 1) |
| `--from-checkpoint <id>` | Fork from a specific checkpoint instead of the current state |

```bash
superserve fork vm_abc123 --count 3
superserve fork vm_abc123 --from-checkpoint cp_x7y8z9
```

---

## `superserve fork tree`

Display the fork tree for a VM, showing parent-child relationships.

```bash
superserve fork tree <vm_id>
```
