## `superserve login`

Authenticate via browser. Credentials stored at `~/.superserve/credentials.json`.

```bash
superserve login
superserve logout
```

---

## `superserve deploy`

```bash
superserve deploy [ENTRYPOINT] [--name NAME] [--port PORT] [--dir PATH] [--json]
```

```bash
superserve deploy agent.py
superserve deploy agent.py --name code-reviewer
superserve deploy server.py --port 8000
superserve deploy              # reads superserve.yaml
```

| Flag | Description |
|------|-------------|
| `ENTRYPOINT` | Script to run (e.g. `agent.py`, `src/main.ts`). Without this, reads `superserve.yaml`. |
| `--name`, `-n` | Agent name (default: directory name) |
| `--port`, `-p` | HTTP proxy mode — forward to this port |
| `--dir` | Project directory (default: `.`) |
| `--json` | Output as JSON |

Supported: `.py`, `.ts`, `.tsx`, `.js`, `.jsx`, `.mjs`, `.cjs`

Dependencies auto-detected from `requirements.txt` or `pyproject.toml`.

## `superserve init`

Generate a `superserve.yaml` config file. Only needed if you want to customize beyond what zero-config provides.

```bash
superserve init [--name NAME]
```

```yaml superserve.yaml
name: my-agent
command: python main.py

# secrets:
#   - ANTHROPIC_API_KEY

# ignore:
#   - .venv
#   - __pycache__
```

If `.env.example` exists, secrets are auto-detected.

## `superserve run`

Start a conversation with a deployed agent. Creates a session automatically.

```bash
superserve run AGENT [PROMPT] [--single] [--json]
```

| Flag | Description |
|------|-------------|
| `PROMPT` | Initial message. Omit for interactive mode. |
| `--single` | Exit after one response |
| `--json` | Raw SSE events as JSON |

```bash
# Interactive multi-turn
superserve run my-agent

# Single prompt, single response
superserve run my-agent "Explain this stack trace" --single
```

## `superserve secrets`

Encrypted environment variables. Values are never returned by the API.

```bash
# Set one or more secrets
superserve secrets set AGENT KEY=VALUE [KEY=VALUE ...]

# List key names (values are never shown)
superserve secrets list AGENT

# Delete a secret
superserve secrets delete AGENT KEY [--yes]
```

## `superserve agents`

```bash
# List all agents
superserve agents list [--json]

# Get agent details
superserve agents get NAME [--json]

# Delete agent and all data
superserve agents delete NAME [--yes]
```

## `superserve sessions`

```bash
# List sessions (optionally filter by agent)
superserve sessions list [AGENT] [--status STATUS] [--json]

# Get session details
superserve sessions get SESSION_ID [--json]

# Resume a previous session
superserve sessions resume SESSION_ID

# End a session (workspace preserved)
superserve sessions end SESSION_ID
```

Session IDs accept short prefixes — `a1b2` matches `a1b2c3d4-e5f6-...`.
