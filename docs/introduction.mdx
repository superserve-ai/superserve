Superserve is a sandbox and VM provider for AI agents and developer tools. It gives you on-demand Firecracker micro-VMs with sub-200ms boot times, full Linux environments, and primitives for checkpoint, rollback, and fork.

## Why Superserve

AI agents and automated workflows need isolated compute environments that boot fast, persist state, and branch execution. Traditional VMs are too slow. Containers lack real isolation. Superserve gives you the best of both: hardware-isolated micro-VMs that feel as fast as containers.

- **Sub-200ms boot**: Firecracker micro-VMs start in under 200 milliseconds. Spin up sandboxes on demand without cold-start delays.
- **Checkpoint and rollback**: Snapshot a VM at any point. Roll back to a previous state in milliseconds. Undo destructive changes instantly.
- **Fork**: Clone a running VM into multiple independent copies. Run parallel experiments from the same starting point.
- **File upload and download**: Push files into and pull files out of VMs programmatically.
- **Command execution**: Run arbitrary commands inside VMs via the API, CLI, or SSH.

## VM State Machine

VMs follow a lifecycle with these states:

```
CREATING --> RUNNING --> STOPPED
               |            |
               v            v
           SLEEPING       DEAD
               |
               v
            RUNNING (wake)
```

| State | Description |
|-------|-------------|
| `CREATING` | VM is being provisioned |
| `RUNNING` | VM is active and accepting commands |
| `STOPPED` | VM is stopped. Can be started again or deleted. |
| `SLEEPING` | VM is suspended to disk. Wake to resume instantly. |
| `DEAD` | VM has been deleted. Terminal state. |

## Quick Example

```bash
# Install and authenticate
bun add -g @superserve/cli
superserve login

# Create a VM
superserve vm create --name my-sandbox --image ubuntu-22.04

# Run a command
superserve exec vm_abc123 "echo hello world"

# Upload a file
superserve files upload vm_abc123 ./script.py /app/script.py

# Checkpoint and fork
superserve checkpoint create vm_abc123 --name baseline
superserve fork vm_abc123 --count 3
```
