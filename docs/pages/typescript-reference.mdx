The `SuperserveClient` exposes four resource clients: `sandboxes`, `exec`, `files`, and `system`. Every method returns a `Promise` and throws a typed error subclass on non-2xx responses.

## Client

```ts
import { SuperserveClient } from "@superserve/sdk"

const client = new SuperserveClient({
  apiKey: "ss_live_...",           // required
  environment?: "production",      // or pass a custom baseUrl
  baseUrl?: "https://...",         // overrides environment
  timeoutInSeconds?: 60,
  maxRetries?: 2,
})
```

## `client.sandboxes`

Lifecycle operations for Firecracker MicroVM sandboxes.

### `listSandboxes()`

Lists all sandboxes for the authenticated team.

```ts
const { sandboxes } = await client.sandboxes.listSandboxes()
```

### `createSandbox(request)`

Creates a new sandbox.

```ts
const sandbox = await client.sandboxes.createSandbox({
  name: "my-sandbox",            // required
  from_snapshot: "snp_...",      // optional — boot from snapshot
  network: { /* ... */ },        // optional network config
})
```

Returns a `SandboxResponse` with `id`, `name`, `status`, `vcpu_count`, `memory_mib`, `ip_address`, `snapshot_id`, `created_at`.

### `getSandbox(sandboxId)`

Fetches the current state of a sandbox.

```ts
const sandbox = await client.sandboxes.getSandbox("sbx_...")
```

### `patchSandbox(sandboxId, request)`

Updates mutable fields on a sandbox (e.g. name).

```ts
await client.sandboxes.patchSandbox("sbx_...", {
  name: "renamed",
})
```

### `pauseSandbox(sandboxId)`

Pauses a running sandbox. Fern takes a snapshot and shuts down the VM. The sandbox can be resumed later.

```ts
await client.sandboxes.pauseSandbox("sbx_...")
```

### `resumeSandbox(sandboxId)`

Resumes a previously paused sandbox from its snapshot.

```ts
await client.sandboxes.resumeSandbox("sbx_...")
```

### `deleteSandbox(sandboxId)`

Permanently deletes a sandbox. Any in-progress execution is terminated.

```ts
await client.sandboxes.deleteSandbox("sbx_...")
```

## `client.exec`

Run commands inside a sandbox.

### `command(request)`

Runs a command and waits for it to complete. Good for short-lived commands.

```ts
const result = await client.exec.command({
  sandbox_id: "sbx_...",
  body: {
    command: "ls -la /",
  },
})

result.stdout     // string
result.stderr     // string
result.exit_code  // number
```

### `commandStream(request)`

Runs a command and streams stdout/stderr events as they're produced. Use for long-running commands or interactive output.

```ts
const stream = await client.exec.commandStream({
  sandbox_id: "sbx_...",
  body: { command: "npm install" },
})

for await (const event of stream) {
  if (event.type === "stdout") process.stdout.write(event.data)
  if (event.type === "stderr") process.stderr.write(event.data)
  if (event.type === "done") console.log("exit code:", event.exit_code)
}
```

## `client.files`

Upload and download files to and from a sandbox. See the generated types in `@superserve/sdk` for full request shapes.

## `client.system`

### `health()`

Health check for the Superserve API itself.

```ts
const health = await client.system.health()
```

## Errors

All errors extend `SuperserveError`. Specific subclasses:

- `BadRequestError` — 400
- `UnauthorizedError` — 401
- `NotFoundError` — 404
- `ConflictError` — 409
- `InternalServerError` — 500
- `SuperserveTimeoutError` — request exceeded `timeoutInSeconds`

```ts
import { SuperserveClient, BadRequestError } from "@superserve/sdk"

try {
  await client.sandboxes.createSandbox({ name: "" })
} catch (err) {
  if (err instanceof BadRequestError) {
    console.error(err.message, err.body)
  } else {
    throw err
  }
}
```
